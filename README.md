## 建站实际配置要多少资源

问题是：1GB 内存够不够？

够用，但要看怎么配置。用宝塔面板装 WordPress，默认配置下 1GB 内存会比较紧张——MySQL 加 Nginx 加 PHP 跑起来大概用掉 600-700MB，剩下的给 WordPress 本身，访问量一大就开始抖。

解决方法：

1. 装 WP Super Cache 或者 WP Rocket 做静态缓存，动态查询变成静态页面吐出去，1GB 内存扛住日常博客访问没问题
2. 调低 MySQL 的 innodb_buffer_pool_size，通常从默认的 128MB 降到 64MB 或 32MB
3. 开 Swap（搬瓦工 KVM 支持），1GB 内存 + 1GB Swap 应急用

要是跑的是 WordPress 电商站（WooCommerce）或者要装多个站点，老老实实上 2GB 版本，省得以后操心。
